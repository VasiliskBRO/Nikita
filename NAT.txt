Часто при использовании IPv4 используется NAT для маскирования несколько адресов под один, так чтобы защитить ЛВС и получить доступ в интернет имея всего один белый адрес такой NAT назваться PAT он множество адресов маскирует под один.
Для его настройки необходимо указать, где локальная сеть (клиентские интерфейсы) и глобальная сеть (интерфейсы куда будут выбрасываться замаскированные адреса).
interface (номер интерфейса в сторону интернета)
ip nat outside
interface (номер интерфейса в сторону клиентов)
ip nat inside
exit
Также потребуется создать правило или правила для маскирования (например, кому и что разрешается) можно разрешить все permit any, а можно лишь некоторые адреса и подсети.
access-list (№ ACL) permit any
access-list (№ ACL) permit ip (подсеть) (обратная маска) any
access-list (№ ACL) permit ip host (ip адрес) any
Далее необходимо создать сам NAT добавив нужное правило, а также указать интерфейс куда необходимо выбрасывать маскированные адреса
ip nat inside source list (№ ACL) interface ethernet (интерфейс в сторону интернета) overload 
Для выхода в интернет можно использовать статический маршрут если не важно откуда и по какой маски, то можно указать откуда угодно и по любой маски, но обязательно необходимо указывать куда в виде шлюза
ip route 0.0.0.0 0.0.0.0 (шлюз)
